## 一、什么是 AI Agent

### 1.1 定义

**AI Agent（人工智能智能体）** 是一种能够**感知环境、自主决策、采取行动**以完成目标的系统。在 LLM 时代，AI Agent 特指以大语言模型为"大脑"，结合记忆、规划、工具调用等能力，能够自主完成多步骤复杂任务的智能系统。

Anthropic 给出一个简洁的定义：

> Agent 是这样一类系统：LLM 动态地决定自己的处理流程、使用什么工具，并对任务结果保持控制权。

### 1.2 Agent 与传统程序、纯 LLM 的区别

| 维度 | 传统程序 | 纯 LLM（Chatbot） | AI Agent |
|---|---|---|---|
| 控制流 | 代码写死，确定执行 | 单轮请求-响应 | LLM 动态决定下一步 |
| 知识来源 | 数据库、硬编码 | 仅预训练参数 | 参数知识 + 外部检索 + 实时工具 |
| 行动能力 | 调 API | 只输出文本 | 调用工具、操作环境、执行代码 |
| 自主性 | 无 | 无 | 有目标导向、可规划、可反思 |
| 状态管理 | 完整可控 | 无状态/单会话 | 短期记忆 + 长期记忆 |

**核心区别**：Agent 的"下一步做什么"由 LLM 在运行时决定，而非由代码预先写死。

### 1.3 Agent 的核心特征

1. **自主性（Autonomy）**：无需人类逐步指令，能自主分解任务、规划路径
2. **反应性（Reactivity）**：能感知环境变化（工具返回、外部事件）并调整行为
3. **主动性（Pro-activeness）**：由目标驱动，不只是被动响应
4. **社交性（Social ability）**：能与其他 Agent 或人类协作（多 Agent 系统）
5. **学习性（Learning）**：能从经验中改进（记忆、反思、自我优化）

## 二、框架对比总表

### 2.1 核心能力对比

| 框架                     | 编排范式       | 语言                | 模型无关         | 上手曲线 | 生产级持久化 | 典型场景          |
|------------------------|------------|-------------------|--------------|------|--------|---------------|
| **LangGraph**          | 图/状态机      | Python/JS         | ✅ 全模型        | 陡    | ⭐⭐⭐⭐⭐  | 复杂有状态工作流      |
| **CrewAI**             | 角色驱动       | Python            | ✅ 全模型        | 缓    | ⭐⭐     | 多 Agent 协作原型  |
| **MS Agent Framework** | 对话+工作流     | Python/.NET       | ✅ 多 provider | 中    | ⭐⭐⭐⭐   | 企业级一体化        |
| **OpenAI Agents SDK**  | Handoff 路由 | Python            | ⚠️ OpenAI 优先 | 最平缓  | ⭐⭐⭐    | 快速验证/标准化生产    |
| **Google ADK**         | 图+协作       | Python/Java/Go/TS | ✅ 多模型        | 中    | ⭐⭐⭐    | 多模态/Google 生态 |
| **Dify**               | 可视化低代码     | Web               | ✅ 多模型        | 极低   | ⭐⭐⭐    | 非技术团队         |
| **Coze**               | 零代码        | Web               | ⚠️ 字节系       | 极低   | ⭐⭐     | C 端应用         |
| **LlamaIndex**         | 数据驱动       | Python            | ✅ 多模型        | 中    | ⭐⭐⭐    | 数据密集型 RAG     |
| **Pydantic AI**        | 类型安全       | Python            | ✅ 多模型        | 低    | ⭐⭐     | 结构化输出场景       |
